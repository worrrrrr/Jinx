## Core Concepts (15 Additional Jinx Intents)

|**Intent**|**Keywords (Thai)**|**Keywords (English)**|**Descriptions**|
|---|---|---|---|
|**configure**|ตั้งค่า, คอนฟิก, กำหนดค่า, ปรับแต่งระบบ|configure, set, setup, config, settings|ใช้สำหรับจัดการ Configuration หรือ Environment Variables ของระบบ|
|**rollback**|ย้อนกลับ, รีเวิร์ต, ถอยกลับ, เอาเวอร์ชันเก่า|rollback, revert, undo, downgrade|ใช้เมื่อระบบเกิดข้อผิดพลาดและต้องการย้อน State กลับไปจุดที่เสถียร|
|**authenticate**|ล็อกอิน, เข้าสู่ระบบ, ยืนยันตัวตน, ใส่รหัส|login, authenticate, sign in, verify identity|ใช้ในกระบวนการ Security Identity Verification และสิทธิ์เข้าถึงระบบ|
|**authorize**|ขอสิทธิ์, เปิดสิทธิ์, อนุญาต, ปลดล็อกสิทธิ์|authorize, grant, permit, allow access|ใช้สำหรับร้องขอหรือปรับแต่งสิทธิ์ในการเข้าถึง Resource (RBAC/ABAC)|
|**audit**|ตรวจสอบย้อนหลัง, ดูประวัติการใช้งาน, ตรวจสอบ Log|audit, review log, inspect history, track|ใช้ตรวจสอบความถูกต้อง ความปลอดภัย และประวัติการทำงานย้อนหลัง|
|**terminate**|ปิดถาวร, ทำลาย, ยกเลิกการติดตั้ง, ลบระบบ|terminate, destroy, kill, uninstall, wipe|ใช้เมื่อต้องการปลดระวาง (Decommission) สคริปต์ ตัวแปร หรือ Infrastructure|
|**synchronize**|ซิงค์ข้อมูล, อัปเดตให้ตรงกัน, ผสานระบบ|sync, synchronize, alignment, update state|ใช้สั่งการให้ Data Node ตั้งแต่สองแหล่งขึ้นไปมีสถานะข้อมูลที่ตรงกัน|
|**migrate**|ย้ายระบบ, ย้ายฐานข้อมูล, ไมเกรต|migrate, relocate, transfer data, move|ใช้สำหรับกระบวนการย้ายโครงสร้างข้อมูลหรือ Schema ไประบบใหม่|
|**pause**|หยุดชั่วคราว, พักไว้ก่อน, เบรก|pause, suspend, hold, freeze|ใช้สั่งการชะลอการทำงานของ Process หรือ Queue โดยไม่ทำลาย State|
|**resume**|ทำงานต่อ, รันต่อ, ปลดล็อก|resume, unpause, continue, restart process|ใช้กู้คืนสถานะกระบวนการที่ถูกสั่ง Pause ให้กลับมาประมวลผลต่อ|
|**mock**|จำลองข้อมูล, เมคข้อมูล, สมมติตัวแปร|mock, stub, fake data, simulate object|ใช้สร้าง Fake Payload สำหรับทดสอบระบบใน [[Conceptual Sandboxing]]|
|**benchmark**|ทดสอบความเร็ว, วัดประสิทธิภาพ, เทสโหลด|benchmark, load test, stress test, profiling|ใช้รันการวัดประสิทธิภาพเชิงตัวเลขของระบบ (Throughput/Latency)|
|**subscribe**|ติดตาม, สมัครรับข้อมูล, ตรวจสอบอีเวนต์|subscribe, listen, follow, register webhook|ใช้ผสานการทำงานกับระบบ Event-Driven เพื่อดักฟัง Event Stream|
|**feedback**|ส่งคำติชม, รายงานปัญหา, แนะนำฟีเจอร์|feedback, report issue, user review, suggest feature|ใช้รับความเห็นผู้ใช้เพื่อนำเข้าสู่กระบวนการปรับปรุงระบบ (Feedback Loop)|
|**escalate**|ขอคุยกับคน, ส่งต่อให้เจ้าหน้าที่, เรียกหัวหน้า|escalate, transfer to human, call agent, higher support|ใช้เมื่อระบบ NLU ไม่สามารถตอบสนองต่อไปได้ และต้องการเปลี่ยนผ่านไปหาคน|
