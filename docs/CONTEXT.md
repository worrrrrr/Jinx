# CONTEXT

## Why JINX Exists

JINX ถูกสร้างขึ้นจากข้อสังเกตว่า LLM เพียงอย่างเดียวไม่ใช่คำตอบของระบบ AI ที่เชื่อถือได้

ปัญหาที่พบจากการทดลองกับโมเดลหลายตัว

* โมเดลบางตัวตอบเร็วแต่ผิด
* โมเดลบางตัวตีความโจทย์เกินความจำเป็น
* โมเดลบางตัวใช้ Token มากเกินไป
* คำตอบเปลี่ยนได้แม้ใช้คำถามเดิม

ดังนั้นเป้าหมายของ JINX ไม่ใช่การสร้าง Chatbot

แต่เป็นการสร้าง AI Runtime ที่มีพฤติกรรมคาดเดาได้และตรวจสอบได้

---

## Core Belief

ความฉลาดไม่ได้มาจาก LLM เพียงอย่างเดียว

ความฉลาดเกิดจาก

* การเข้าใจปัญหา
* การเลือกวิธีแก้ปัญหา
* การใช้เครื่องมือที่เหมาะสม
* การเข้าถึงข้อมูลที่ถูกต้อง

---

## Ground Truth

ข้อมูลภายในระบบคือ Ground Truth

ตำแหน่งหลักของความรู้

```text id="r5"
data/knowledge/
```

ข้อมูลใน Vault มีความสำคัญมากกว่าความเห็นของ LLM

เมื่อข้อมูลใน Vault และ LLM ขัดแย้งกัน

ให้เชื่อข้อมูลใน Vault ก่อน

---

## Role of LLM

LLM ไม่ใช่ศูนย์กลางของระบบ

LLM เป็นส่วนเสริม

ตัวอย่างการใช้งาน

* อธิบายผลลัพธ์
* ตรวจสอบคำตอบ
* เปรียบเทียบแนวคิด
* สรุปข้อมูล

LLM ไม่ควรแทนที่ Tool

LLM ไม่ควรแทนที่ Ground Truth

---

## Why Not Agent First

Agent Framework จำนวนมากให้ LLM เป็นผู้ควบคุมทุกอย่าง

แนวทางของ JINX แตกต่างออกไป

JINX ให้

* Rules
* Knowledge
* Tools

เป็นผู้ควบคุมระบบ

LLM เป็นเพียงผู้ช่วย

เหตุผลคือระบบที่อิง Rules และ Tools สามารถตรวจสอบและทดสอบได้ง่ายกว่า

---

## Knowledge Before Intelligence

หากไม่มีความรู้ที่ถูกต้อง

ความฉลาดไม่มีประโยชน์

ดังนั้นระบบจึงให้ความสำคัญกับ

1. Knowledge
2. Tools
3. Workflow
4. Language

ตามลำดับ

---

## Long-Term Vision

เป้าหมายระยะยาวไม่ใช่การสร้าง Chatbot

แต่คือการสร้าง Runtime ที่สามารถ

* อ่านข้อมูล
* เข้าใจปัญหา
* วางแผน
* ใช้เครื่องมือ
* ตรวจสอบผลลัพธ์
* ตอบกลับ

โดยมี Ground Truth ที่ตรวจสอบได้เสมอ
