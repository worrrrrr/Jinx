# RULES

## Purpose

เอกสารนี้เก็บกฎหลักของระบบ

กฎเหล่านี้มีความสำคัญมากกว่าโครงสร้างไฟล์หรือรายละเอียดการ Implement

หากมีการเปลี่ยนแปลงระบบในอนาคต ต้องพยายามรักษากฎเหล่านี้ไว้

---

# 1. Knowledge Is Ground Truth

ข้อมูลใน

data/knowledge/

ถือเป็นแหล่งความจริงหลักของระบบ

เมื่อข้อมูลจากภายในและข้อมูลจาก LLM ขัดแย้งกัน

ให้เชื่อข้อมูลจาก Knowledge ก่อน

---

# 2. Tools Over Guessing

งานที่สามารถคำนวณหรือพิสูจน์ได้

ต้องใช้ Tool

ห้ามเดาคำตอบ

ตัวอย่าง

* คณิตศาสตร์
* การค้นหา
* การวิเคราะห์ข้อมูล
* การสร้างไฟล์

---

# 3. LLM Is Optional

ระบบต้องสามารถทำงานได้แม้ไม่มี LLM

LLM เป็นเพียงส่วนเสริม

ไม่ใช่ศูนย์กลางของระบบ

---

# 4. Deterministic First

Input เดียวกัน

ควรได้ผลลัพธ์เหมือนเดิม

หลีกเลี่ยงการสุ่มหรือการเดาที่ไม่จำเป็น

---

# 5. Explicit Workflow

ทุกคำตอบต้องสามารถอธิบายได้ว่า

* รับรู้อะไร
* วางแผนอย่างไร
* ใช้ Tool อะไร
* ได้ผลลัพธ์จากไหน

---

# 6. Separation of Responsibilities

แต่ละ Component มีหน้าที่ชัดเจน

Perception

* เข้าใจข้อมูล

Reasoning

* เลือกแนวทาง

Execution

* เรียกใช้งาน

Response

* สื่อสารผลลัพธ์

ห้ามรวมหน้าที่จนเกิดความสับสน

---

# 7. Knowledge Before Intelligence

หากไม่มีข้อมูลที่ถูกต้อง

ความฉลาดไม่มีความหมาย

ระบบให้ความสำคัญกับ

Knowledge
↓
Tools
↓
Workflow
↓
Language

ตามลำดับ

---

# 8. Testability

ทุก Tool ต้องสามารถทดสอบได้

ทุกผลลัพธ์ควรตรวจสอบย้อนกลับได้

---

# 9. Simplicity First

เลือกวิธีที่ง่ายที่สุดก่อน

เพิ่มความซับซ้อนเมื่อมีเหตุผลชัดเจนเท่านั้น

หลีกเลี่ยง Framework หรือ Layer ที่ไม่จำเป็น

---

# 10. Human Control

มนุษย์เป็นผู้ตัดสินใจสุดท้าย

ระบบมีหน้าที่ช่วยวิเคราะห์ วางแผน และดำเนินการ

ไม่ใช่แทนที่การตัดสินใจของผู้ใช้
